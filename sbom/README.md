# SBOM artefacts

CycloneDX SBOMs for the backend (Python) and frontend (Node) are generated by `scripts/sbom.sh`:

```
./scripts/sbom.sh
```

Outputs:

- `backend-sbom.json` — latest backend dependency graph
- `frontend-sbom.json` — latest frontend dependency graph
- `*-sbom-<timestamp>.json` — historical runs (all timestamped copies are
  git-ignored so the directory stays lean; check them in only when you
  need an auditable record of a specific release)

The scripts install the generator tooling lazily:

- Backend: `cyclonedx-bom` into the backend virtualenv
- Frontend: `@cyclonedx/cyclonedx-npm` via `npx`

Output format is JSON by default. Tools like
[Dependency-Track](https://dependencytrack.org/) ingest these directly.
